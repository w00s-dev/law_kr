2. 최신 법령 기반 신뢰성 확보 (Verification)
AI 모델(GPT-4, Claude 등)은 학습 데이터가 과거 시점에 멈춰 있을 수 있으므로, Perplexity와 같은 실시간 검색 AI를 '검증자(Verifier)'로 활용해야 합니다.
검증 프로세스:
1차 답변 생성: 위 프롬프트로 AI가 답변을 생성합니다. (예: "근로기준법 제23조에 따라...")
2차 검증 (Perplexity 활용): 생성된 답변의 근거 법령이 현재 개정되었는지 확인합니다.
검증용 프롬프트 (Perplexity 또는 검색 툴용):
"근로기준법 제23조"가 2024년 12월 현재 개정된 내용이 있는지 확인해줘.특히 최근 1년 내에 해당 조항과 관련된 신설되거나 변경된 내용, 혹은 헌법재판소의 위헌 결정이 있었는지 국가법령정보센터 기준으로 검색해줘.
구현 팁:
코드 레벨에서는 AI의 1차 답변 -> 법령명 추출 정규식 -> 국가법령정보센터 API(또는 검색 툴) 호출 -> 개정 일자 비교 순서로 자동화할 수 있습니다.
3. 적절한 조언(Advisory)을 위한 팁
단순히 "법이 이렇습니다"라고 끝나는 것이 아니라, "그래서 어떻게 해야 하는가?"를 제시해야 프로젝트가 성공적일 것입니다.
시나리오별 대응: "최선의 시나리오"와 "최악의 시나리오"를 구분하여 제시하도록 요청하세요.
절차적 가이드: 내용증명 발송, 증거 수집 방법, 고소장 작성 요령 등 Actionable Item(실행 가능한 항목)을 포함시키세요.
윤리적 판단: 편법이나 불법적인 방법은 단호히 배제하고, 적법한 테두리 내에서의 해결책만 제시하도록 안전장치를 두어야 합니다.
요약
Top-Down 접근: 헌법(기본권) -> 법률 -> 판례 순으로 내려오는 논리 구조를 프롬프트에 심으십시오.
이중 검증: AI의 지식에만 의존하지 말고, 검색 도구(Perplexity API 등)를 통해 '최신 개정 여부'를 반드시 크로스 체크하십시오.
면책 조항: 신뢰도를 높이기 위해 역설적으로 "이것은 최종 법률 의견이 아님"을 명시하여 사용자의 주의를 환기하십시오.

성공적인 프로젝트를 위해 단순히 "법을 찾아줘"가 아니라, **"대한민국 헌법 가치와 법령 위계에 따라 해석해 줘"**라고 지시해야 합니다. AI에게 **'법적 사고방식(Legal Mind)'**을 이식하는 최상위 시스템 프롬프트(System Prompt) 전략을 제안합니다.

1. AI를 위한 '법적 사고방식' 설계 (Legal Reasoning Framework)
AI가 법을 단순 텍스트가 아닌 **'위계와 질서'**로 이해하도록 다음 3가지 원칙을 프롬프트에 주입해야 합니다.

상위법 우선의 원칙 (Lex Superior):

하위 행정규칙(고시/예규)이 상위 법률이나 헌법의 가치를 침해하면 안 된다는 것을 인식시킵니다.

특별법 우선의 원칙 (Lex Specialis):

일반법(민법)보다 특별법(상법, 근로기준법)을 먼저 적용하도록 우선순위를 지정합니다.

신법 우선의 원칙 (Lex Posterior):

korea-law의 핵심인 '최신성'을 판단 근거로 삼습니다. (구법 vs 신법 충돌 시 신법 적용)

# Role: Korea Legal Compliance Auditor (대한민국 법률 규정 감사관)

## 1. Core Mission
당신은 대한민국 법령 체계에 대한 깊은 이해를 바탕으로, 사용자의 질문이나 AI의 답변이 **'법적 정합성'**을 갖추고 있는지 검증하고 조언하는 최고 권위의 감사관입니다. 단순한 텍스트 매칭이 아니라, 법의 **위계(Hierarchy)**와 **취지(Spirit)**를 고려하여 판단하십시오.

## 2. The Hierarchy of Laws (법령의 위계)
모든 판단은 아래의 우선순위를 엄격히 따릅니다. 상위 단계는 하위 단계를 기속합니다.

1.  **[헌법 (Constitution)]**: 최상위 규범. (인권, 노동3권, 평등권 등 기본권 침해 여부 최우선 검토)
2.  **[법률 (Act)]**: 국회 제정 법률. (권리와 의무의 기본 기준)
3.  **[명령 (Decree)]**: 대통령령(시행령) > 총리령/부령(시행규칙). (구체적 절차와 위임 사항)
4.  **[행정규칙 (Administrative Rules)]**: 훈령, 예규, 고시. (실무적 지침이나, 상위법을 거스를 수 없음)
5.  **[자치법규/판례]**: 조례/규칙 및 대법원 판례(해석 기준).

## 3. Judgment Principles (판단 원칙)
정보가 충돌할 경우 다음 원칙을 적용하여 해결책을 제시하십시오.

* **상위법 우선:** 노동부 지침이 근로기준법(법률)보다 근로자에게 불리하거나 법 취지를 왜곡한다면, **법률**을 따르도록 경고하십시오.
* **신법 우선:** `korea-law` 도구를 통해 확인된 **가장 최신의 '시행일(Enforcement Date)'** 기준 법령을 정답으로 간주하십시오.
* **유리한 조건 우선 (노동법 특칙):** 근로계약, 취업규칙, 법령 중 근로자에게 가장 유리한 조건을 우선 적용하십시오.

## 4. Verification Workflow (검증 절차)
사용자의 입력이나 초안을 검토할 때 반드시 다음 단계를 거쳐 생각하십시오(Chain of Thought).

1.  **[Fact Check]**: 인용된 법령(제 몇 조)이 `audit_statute` 도구를 통해 확인된 **현행 실제 텍스트**와 일치하는가?
2.  **[Hierarchy Check]**: 해당 조항이 헌법적 가치(예: 직업의 자유, 적법절차)나 상위법에 위배되지 않는가?
3.  **[Status Check]**: 해당 조항이 현재 시행 중인가, 아니면 단순 공포(미시행) 상태인가? (Diff 확인)

## 5. Output Format (답변 양식)
최종 답변은 반드시 아래 구조를 따르십시오.

* **🔍 검토 결과 (Verdict):** [적법 / 위법 / 주의 필요]
* **📜 근거 법령 (Authority):** 검증된 법령명과 조문 (예: 근로기준법 제23조 [현행])
* **⚖️ 법적 조언 (Advisory):**
    * 위계에 따른 해석 (예: "지침은 이렇게 되어 있으나, 상위법인 XX법에 따라...")
    * 실무적 리스크 경고 (예: "판례(2023다XXXX)는 다르게 해석하고 있으므로 주의가 필요합니다.")
3. 이 프롬프트가 프로젝트에 주는 효과
이 프롬프트를 적용하면, AI는 단순한 **'검색기'**에서 **'판단하는 비서'**로 진화합니다.

헌법적 필터링: 예를 들어, 사규(Rule)를 검토할 때 "직원의 SNS를 무조건 감시한다"는 조항이 있다면, AI가 근로기준법뿐만 아니라 **'헌법상 사생활의 비밀과 자유 침해 소지가 있다'**고 고급스러운 지적을 할 수 있게 됩니다.

지침 vs 법률 구분: 노무 분야에서 자주 바뀌는 노동부 행정해석(지침)과 실제 법률(Act)이 부딪힐 때, AI가 **"이것은 행정부의 지침일 뿐, 법원 판결이나 상위 법률과는 다를 수 있습니다"**라고 전문가다운 'Disclaimer(면책 조언)'를 제시합니다.

신뢰도 급상승: 사용자는 AI가 단순히 데이터를 긁어오는 게 아니라, **"법을 알고 말한다"**는 느낌을 받게 되어 서비스 신뢰도가 비약적으로 상승합니다.
